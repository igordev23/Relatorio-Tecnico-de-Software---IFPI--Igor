\section{Considerações Finais}

O presente trabalho alcançou o objetivo de projetar e implementar um sistema web automatizado para emissão, gestão e autenticação de certificados, substituindo processos manuais e soluções improvisadas por uma plataforma profissional, segura e escalável. O sistema foi desenvolvido seguindo a abordagem de Produto Mínimo Viável (MVP), priorizando as funcionalidades essenciais para a operação — templates configuráveis, emissão de certificados com código único de verificação, geração de PDF, dashboard com indicadores gerenciais e portal público de validação via QR Code.

A arquitetura em duas aplicações independentes (frontend SPA e backend API REST) mostrou-se adequada ao cenário de SaaS multi-tenant, permitindo que cada camada seja desenvolvida, testada e implantada de forma independente. No frontend, a adoção de React 19 com TypeScript, Tailwind CSS e TanStack Router proporcionou uma interface responsiva e de fácil manutenção, enquanto a arquitetura em camadas com ViewModels facilitou a testabilidade da lógica de apresentação. No backend, a arquitetura limpa com inversão de dependências garantiu o isolamento das regras de negócio, possibilitando a substituição de implementações de infraestrutura sem impacto na lógica de domínio. A escolha do Next.js exclusivamente como servidor de API demonstrou ser uma alternativa viável a frameworks tradicionais como Express, reduzindo a configuração necessária sem perder flexibilidade.

Os resultados dos testes confirmam a qualidade da implementação: 276 testes unitários distribuídos entre frontend (108 testes, 98,11\% de cobertura) e backend (168 testes, 97,39\% de cobertura), ambos superando o limiar mínimo de 75\% estabelecido para o projeto. O teste end-to-end com Playwright validou o fluxo crítico de ponta a ponta, desde a criação de um template até a verificação pública do certificado emitido. A integração contínua via GitHub Actions assegura que todo código enviado à branch principal seja automaticamente validado antes da implantação.

A infraestrutura em nuvem, com frontend hospedado como Static Site e backend como Web Service na plataforma Render, demonstrou ser suficiente para o porte do MVP, oferecendo deploys automáticos, certificados SSL gratuitos e rollback com um clique. O banco de dados PostgreSQL gerenciado pelo Supabase atendeu aos requisitos de integridade referencial e consultas complexas inerentes ao modelo relacional do sistema.

\subsection{Limitações}

Apesar dos resultados alcançados, algumas limitações devem ser registradas. O módulo de autenticação e controle de acesso (RF001--RF007) não foi completamente testado — os testes existentes cobrem apenas o middleware de autenticação, deixando o fluxo completo de login, redefinição de senha e gerenciamento de sessões pendente de validação automatizada. Testes de integração com o banco de dados Supabase não foram realizados, uma vez que os repositórios foram testados com o cliente Supabase mockado. O teste end-to-end cobre exclusivamente o fluxo principal (happy path), sem abordar cenários de erro como template ausente ou CPF inválido. Testes de acessibilidade e desempenho (carga) tiveram cobertura limitada. Ademais, funcionalidades previstas no escopo inicial, como importação em lote via CSV/Excel, envio automático de e-mail e emissão via formulário público, foram postergadas para versões futuras por não serem essenciais ao MVP.

\subsection{Trabalhos Futuros}

Com base nas limitações identificadas e nas necessidades da empresa parceira, as seguintes extensões são sugeridas para versões futuras do sistema: (i) implementação do módulo de importação em lote a partir de arquivos CSV e Excel (.xlsx) com processamento em fila assíncrona; (ii) desenvolvimento do módulo de notificação por e-mail com suporte a variáveis dinâmicas e reenvio automático em caso de falha; (iii) criação de um formulário público de inscrição que permita ao participante solicitar seu próprio certificado; (iv) expansão da cobertura de testes com cenários de integração, desempenho e acessibilidade; (v) implementação de relatórios gerenciais exportáveis (PDF e Excel); (vi) suporte a múltiplos idiomas na interface e nos certificados; e (vii) integração com sistemas acadêmicos externos via API para emissão automatizada a partir de listas de aprovados.
